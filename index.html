Обычная документация кода

  • Время изучения документации
  • Нужно перечитывать
  • Написал код - измени документацию
  • Чаще всего - неактуальная
  • Вне кода
  • Хорошо, если она есть

  • Ненадёжные

  • Трудно поддерживать

  • Иногда неактуальны

Решение - Типизация проекта

«Если это выглядит как утка, плавает как утка и крякает как утка, то, вероятно, это утка (какая разница, что это на самом деле)»

Почему TypeScript?

  • Почти единый синтаксис с JavaScript
  • Богатая экосистема
  • Большое комьюнити
  • Большие компании используют
  • Простота освоения

Создаем машину

function createCar(
  fuel: boolean,
  model: number,
  owner: string): Car {
  let car;
  /* business logic */
  return car;
}

TypeScript типы

  • Надёжные
  • Всегда актуальные
  • Имеют чётко определенный синтаксис
  • Находятся в коде
  • Поддержка IDE
  • Расширяют обычный JS
  • Нужно время для изучения синтаксиса
  • Написание дополнительного кода

Unit тесты? А может сначала TypeScript?

Когда использовать

  • Нужно быстро и точно понимать код
  • Это важно
  • Это сложно
  • Нужно рефакторить

Не нужно если

  • Это временное решение
  • Прототип
  • Это простое и маленькое
  • Вы единственный разработчик

Каковы самые слабые места Vue?

Сейчас настройка TypeScript во Vue проекте очень тривиальна

нужно всего лишь...
  1. Настроить tsconfig.json
  2. Настроить ваш сборщик под импорт TS (ts-loader в Webpack)
  3. Настроить импорт однофайловых компонентов *.vue (vue-shims.d.ts)
  4. Добавить в зависимости для декораторов
    • vue-class-component
    • vue-property-decorator

А может проще будет сову нарисовать?
